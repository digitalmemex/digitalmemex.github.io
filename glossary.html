Welcome to the DIGITAL MEMEX glossary.

  • Application: Programm that visualizes data in a particular way
  • Plugin: Extra functionality of DeepaMehta backend
  • Backend: the data access layer of an applicacion
  • Frontend: The presentation layer of an application
  • REST: REpresentational State Transfer is an architectural style, and an approach to communications that is often used in the development of Web services
  • API: a set of functions and procedures that allow the creation of applications which access the features or data of another application.
  • Graph database: a database where the structure of the data is a graph/map
  • Instance
  • Workspace: topmost container for our content
  • Topicmap: it shows network structures, consisting of topics(the little balls) and associations(the lines)
  • Association: represents relationships between topics
  • Topic: represents any concept, from people, countries, and organizations to software modules, individual files, and events
  • Type: are simple or complex structures describing how information is represented in a machine
  • Topic type: represent items of certain type
  • Association Type: represent relations between items
  • Data type: classification of various types of data(in DeepaMehta, Number, Boolean, Text, HTML, Composite)
  • Composite: a data type that allows to store more complex types of data
  • Composition Definition: establishes a strong, direct dependence of the child to the parent type
  • Aggregation Definition: establishes a loose dependence of the child to its parent type
